# 主要创新点

本作品在系统架构、交互设计、功耗管理和云端协同等方面提出了多项创新，具体如下：

在感知调度层面，提出"以人脸检测结果作为系统资源动态分配触发器"的调度策略。系统待机时摄像头以低帧率（5 FPS）运行轻量级检测模型，仅维持最低限度的视觉感知管线；当检测到有效用户人脸时，系统在 300ms 内完成从低功耗感知模式到全功能交互模式的瞬间切换，暂停摄像头管线以释放 CPU 算力和 MIPI-CSI 总线带宽给 LCD 渲染和语音处理，与传统常亮方案相比待机功耗降低约 80%，同时保证用户靠近时的即时响应。在系统架构层面，设计"端侧实时智能 + 云端运营管理"的双层协同架构，将实时性要求高的人脸识别、语音交互、触控响应留在设备端完成，将库存监控、销售分析、远程管理等运营管理任务通过微信小程序实现云端化，小程序通过 14 个 RESTful API 端点与 ESP32 设备通信，运营人员无需到现场即可完成补货决策和经营分析，同时设备端的离线能力保证了网络不佳环境下的基本交易功能。在交互设计层面，将触摸、语音、视觉三种输入方式通过事件队列和回调机制解耦，主循环按优先级轮询处理，触摸响应延迟约 50ms，语音识别延迟约 600ms，视觉检测延迟约 300ms，用户可根据个人习惯和场景自由选择交互方式，松耦合设计使得新增交互方式只需添加新的事件源而无需修改主循环逻辑，LVGL 图形界面采用五状态有限状态机（商品选择、订单确认、支付、出货、感谢）管理各界面切换，通过互斥锁保证多任务环境下的线程安全。在数据管理层面，用户注册、商品购买、日志存储、人脸识别全流程在设备端闭环完成，不依赖公网和云服务，SD 卡以 CSV 格式记录每次购买的完整上下文（用户 ID、时间戳、商品名称、支付金额），人脸数据库和购买日志可直接导入 Excel 进行数据分析，未知人脸自动分配递增 ID 并入库特征向量，消除了传统方案中预注册的繁琐流程，使其适用于网络覆盖不佳的地下车库、偏远地区等场景。在功耗优化层面，设计完整的"运行→空闲检测→深度睡眠→触摸唤醒→恢复运行"低功耗闭环，通过精细的硬件资源管理实现微安级待机功耗，深度睡眠功耗约 3mA 相比待机模式降低 97%，触摸唤醒成功率大于 95%、响应延迟小于 500ms，特别值得注意的是在进入睡眠前正确释放 ISP 硬件句柄（文件作用域静态变量）以保证唤醒后摄像头正常工作，同时仅释放 LCD 显示资源而保留 XPT2046 触摸控制器和 SPI 总线以支持触摸唤醒，配置 GPIO 2 输入上拉并等待触摸释放后再进入睡眠以防止误唤醒。在云端协同层面，微信小程序根据登录角色自动切换用户端和管理员端界面视图，用户端聚焦个性化推荐和购物体验，管理员端聚焦 KPI 仪表盘和库存管理，通过密码认证实现角色切换，个性化推荐基于用户历史购买记录在小程序端实现协同过滤，检测到老用户时自动推送常购商品以缩短选购时间。在视频传输层面，利用 ESP32-P4 内置硬件 JPEG 编码器实现 RGB565 帧到 JPEG 的高效转换，配合 HTTP multipart 协议实现 MJPEG 实时视频流，硬件编码不占用 CPU 算力相比软件编码 CPU 占用降低 90% 以上，MJPEG 流可直接在浏览器中播放无需额外解码器，8 FPS 帧率在保证流畅度的同时将网络带宽控制在约 200KB/s 适合 WiFi 环境下的实时监控。

以上创新点覆盖了感知调度、系统架构、交互设计、数据管理、功耗优化、云端协同和视频传输七个维度，形成了完整的"端侧智能 + 云端管理"解决方案，在保证离线运行能力的同时提供了现代化的远程管理体验。
